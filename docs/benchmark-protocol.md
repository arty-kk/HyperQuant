# Benchmark protocol

The benchmark harness is designed to compare routes, not to create a universal scoreboard.

## Included reports

- `route-benchmark`: route-level trade-offs on synthetic vector and structured context workloads.
- `resident-benchmark`: resident/session, artifact/session, and slice-read metrics for the resident tier.
- `capacity-example`: illustrative node-capacity planning under a fixed RAM budget.

## Rules

- all compared routes run on the same workload in the same process;
- encode and decode timings are reported separately;
- the resident-tier path includes both resident bytes and artifact bytes;
- checked-in evidence is regenerated by `scripts/build_proof_pack.py`.

## What to compare

The most useful comparisons are:

- vector route vs dense rotation baseline,
- context route vs conservative fallback eligibility,
- resident/session vs baseline resident/session,
- capacity under the same RAM budget.

That is more decision-useful than comparing one headline ratio in isolation.
