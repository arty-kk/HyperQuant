# Support

## Use GitHub Issues for

- reproducible bugs,
- feature requests,
- documentation gaps,
- pilot feedback tied to specific workloads.

## Use private channels for

- security reports,
- proprietary trace discussions,
- sensitive deployment details.

## What helps maintainers respond faster

- exact command used,
- input shape and dtype,
- expected vs actual behavior,
- benchmark or resident-plan output when relevant.
